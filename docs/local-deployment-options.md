# 로컬 버전 배포 전략 문서

> 작성일: 2026-04-20  
> 대상: 보험설계사 PC에 직접 설치해서 사용하는 로컬 버전

---

## 핵심 결론

**Vercel(클라우드)과 로컬 버전을 병행 운용하는 하이브리드 전략 권장**

| 용도 | 방식 |
|------|------|
| 샘플/테스트 | Vercel Hobby (무료, 4.5MB 제한) |
| 소규모 실무 | Vercel Pro ($20/월, 대용량 처리) |
| 로컬 실무 | Python FastAPI 로컬 서버 (무제한, 무료) |
| 완성형 배포 | Electron .exe 패키지 (설치 후 더블클릭) |

---

## 로컬 버전 옵션 3가지 상세 비교

### 옵션 1: Python FastAPI 로컬 서버 ← 개발 중 권장

**작동 방식**
```
설계사 PC에서 python run.py 실행
→ 백그라운드에서 서버 시작
→ 브라우저가 자동으로 localhost:8000 열림
→ 웹처럼 사용
```

**장점**
- pdfplumber 그대로 사용 → 테이블 추출 정확도 최고
- 파일 크기 제한 없음 (PC 용량만큼)
- 처리 시간 제한 없음
- 개인정보 외부 서버 전송 없음
- 개발 속도 빠름

**단점**
- 설계사가 Python 설치해야 함
- `python run.py` 명령어 실행 필요 (일반인에게 어색)
- PC마다 환경이 달라서 "내 컴퓨터에선 안 돼요" 문제 발생 가능

**필요 설치 항목**
```
1. Python 3.11+
2. pip install -r requirements.txt
3. python run.py
```

**적합 대상**: 개발자 본인, 또는 어느 정도 기술에 익숙한 설계사

---

### 옵션 2: Docker 로컬 컨테이너

**작동 방식**
```
Docker Desktop 설치
→ docker run 명령어 한 줄 실행
→ 브라우저에서 localhost:8000 접속
→ 사용
```

**장점**
- 환경 완전 격리 → "내 컴퓨터에선 안 돼요" 문제 없음
- Python 직접 설치 불필요
- 버전 관리 쉬움

**단점**
- Docker Desktop 설치 필요 (약 500MB, 설정 필요)
- 일반인에게 Docker 자체가 생소함
- 메모리/CPU 사용량 높음

**적합 대상**: IT 부서가 있는 보험대리점, 기술 이해도 있는 설계사

---

### 옵션 3: Electron 데스크탑 앱 (.exe) ← 완성형

**작동 방식**
```
setup.exe 다운로드 → 더블클릭 설치
→ 바탕화면 아이콘 더블클릭
→ 앱 창 열림
→ PDF 드래그&드롭 → 결과 확인
```

**장점**
- 설치 이후는 일반 프로그램과 동일
- 카카오톡 수준으로 쉬운 사용성
- 인터넷 없이 작동
- 개인정보 완전 로컬 처리

**단점**
- 개발 복잡도 높음 (Next.js + Python 번들링)
- Windows/Mac 각각 빌드 필요
- 업데이트 배포 별도 구현 필요 (자동 업데이트 로직)
- 빌드 파일 크기 약 150~300MB

**기술 스택**
```
Electron (앱 껍데기)
  └── Next.js (UI)
  └── Python (내장 인터프리터 또는 PyInstaller 번들)
        └── pdfplumber (PDF 파싱)
        └── openpyxl (엑셀 생성)
```

**적합 대상**: 일반 보험설계사 최종 배포용

---

## 하이브리드 아키텍처 (권장)

```
[Next.js UI] ← 설정에서 서버 선택 가능
      │
      ├──── 클라우드 모드 ──→ [Vercel API]
      │                           (인터넷 필요, 4.5MB 제한)
      │
      └──── 로컬 모드 ────→ [localhost:8000 (FastAPI)]
                                (인터넷 불필요, 무제한)
```

UI에 서버 선택 토글을 넣으면 같은 화면으로 두 모드 모두 사용 가능.

## 개발 로드맵 재정의

```
Phase 1 (2~3주)
├── Vercel + Node.js(unpdf) 로 샘플 MVP
├── 기능 검증: 기본 PDF 처리 작동 확인
└── 배포: Vercel Hobby (무료)

Phase 2 (1~2주)  
├── Python FastAPI 로컬 서버 추가
├── pdfplumber로 정확도 향상
├── UI에 로컬/클라우드 전환 기능
└── 실무 설계사 1~2명 파일럿 테스트

Phase 3 (2~4주, 선택)
├── Electron .exe 패키지 빌드
├── Windows 설치 파일 생성
├── 자동 업데이트 기능
└── 설계사 전체 배포
```

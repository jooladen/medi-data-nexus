# 초등학생도 이해하는 Vercel PDF 처리 설명서

> 기술 문서가 어려운 분들을 위한 쉬운 버전  
> 보험설계사 도구를 만들기 전에 꼭 읽어보세요

---

## 🏠 Vercel이 뭔가요?

**Vercel = 인터넷 위에 있는 컴퓨터 빌려주는 가게**

우리가 프로그램을 만들면, 그걸 인터넷에 올려야 다른 사람이 쓸 수 있어요.  
그 "인터넷에 올리는 공간"을 Vercel이 제공해줍니다.  
마치 편의점처럼 — 무료로도 쓸 수 있고, 돈 내면 더 좋은 서비스를 받을 수 있어요.

---

## ☁️ 서버리스(Serverless)가 뭔가요?

**서버리스 = 필요할 때만 켜지는 컴퓨터**

일반 서버는 항상 켜져 있어요. (전기 계속 나감)  
서버리스는 손님이 올 때만 켜졌다가, 손님이 가면 꺼져요. (전기 아낌)

### 서버리스의 장점
- 쓴 만큼만 돈 냄 (기본은 무료)
- 설치/관리 불필요

### 서버리스의 단점 ← **이게 문제**
- **켜지는 데 시간이 걸림** (콜드스타트)
- **오래 켜져 있을 수 없음** — 시간 제한 있음
- **파일을 잠깐만 보관** — 요청 끝나면 지워짐
- **받을 수 있는 파일 크기 제한** 있음

---

## 🐍 pdfplumber가 뭔가요?

**pdfplumber = PDF에서 표(테이블)를 뽑아내는 파이썬 도구**

PDF 파일 안에 있는 표를 자동으로 읽어서 숫자와 글자로 변환해주는 프로그램이에요.  
보험 PDF처럼 복잡한 표가 있을 때 가장 정확하게 읽어주는 도구입니다.

**왜 좋냐면**: 내원일수, 투약일수, 처방 코드 같은 표를 정확하게 읽음  
**왜 문제냐면**: 서버리스 환경에서 불안정 (아래에서 자세히 설명)

---

## ⚠️ 서버리스에서 PDF 처리하면 생기는 불편한 점 5가지

### 불편한 점 1️⃣ : 파일 크기 제한 — **4.5 MB 벽**

```
🧱🧱🧱🧱🧱🧱🧱🧱🧱🧱 ← 이 벽보다 큰 파일은 못 들어옴
      4.5 MB 제한
```

Vercel에서 받을 수 있는 파일 크기가 최대 **4.5 MB**예요.

- 일반 보험 PDF 2~3페이지: 약 0.5~1 MB → ✅ 통과
- 보험 PDF 20~30페이지: 약 2~4 MB → ✅ 대부분 통과
- **80페이지 이상 PDF: 5~15 MB → ❌ 막힘!**

**해결책**: Vercel Blob이라는 별도 저장소를 쓰면 됨 (약간 설정 추가 필요)

---

### 불편한 점 2️⃣ : 시간 제한 — **10초 타임아웃**

```
⏱️ 0초 ──────────────── 10초 ──X (강제 종료!)
          처리해야 하는 시간
```

Vercel 무료 플랜은 **10초 안에 처리 못 하면 강제로 꺼버려요.**

- 2~3페이지 PDF 처리: 약 1~3초 → ✅ 괜찮음
- 20페이지 PDF 처리: 약 5~8초 → ⚠️ 빠듯함
- **80페이지 이상 PDF 처리: 10~30초 예상 → ❌ 시간 초과 에러**

**해결책**: 돈 내는 Pro 플랜으로 업그레이드 → 5분까지 가능

---

### 불편한 점 3️⃣ : pdfplumber 불안정

```
🐍 pdfplumber: "나는 내 컴퓨터에서만 잘 작동해요"
☁️ Vercel: "우리 환경은 달라요... 모르겠어요"
```

pdfplumber는 원래 일반 컴퓨터에서 쓰라고 만든 도구예요.  
서버리스 환경(Vercel)에서 제대로 작동하는지 **공식 테스트 결과가 없어요.**  
실제로 쓰다가 갑자기 에러가 날 수 있고, 원인 파악도 어려워요.

---

### 불편한 점 4️⃣ : 진행률 표시 불가

```
일반 서버: "30% 완료..." → "60% 완료..." → "100% 완료!"
서버리스:  무응답... 무응답... 무응답... "완료!" (또는 에러)
```

서버리스는 처리 중에 중간 보고를 못 해요.  
80페이지 PDF 처리하는 동안 화면이 그냥 빙글빙글만 돌고 있어요.  
사용자는 잘 되고 있는지 실패한 건지 알 수가 없어요.

**해결책**: 별도로 진행률 확인 기능을 만들어야 함 (추가 개발 필요)

---

### 불편한 점 5️⃣ : 파일이 처리 후 사라짐

```
요청 들어옴 → PDF 처리 → 결과 반환 → 파일 삭제됨 (자동)
```

서버리스는 작업이 끝나면 임시 파일을 모두 지워요.  
이력을 저장하려면 따로 데이터베이스를 써야 해요.

| 저장 용량 | 500 MB | $0.023/GB/월 (약 33원/GB/월) |
| 파일 전송 | 1 GB/월 | $0.10/GB (약 140원/GB) |

> **현실적 비용**: 보험설계사 1~2명 소규모 사용 기준 → 월 $20~25 예상

## 📊 80페이지 이상 PDF, 얼마나 힘든가요?

### 일반 PDF (5페이지 이하)
```
[업로드] ──2초──→ [처리 완료] ✅
비용: 무료 플랜으로 가능
```

### 중간 PDF (20~30페이지)
```
[업로드] ──5~8초──→ [처리 완료] ⚠️ 빠듯함
비용: 무료 플랜 위험 → Pro 권장
```

### 대용량 PDF (80페이지 이상)
```
[파일 크기 5~15 MB] → 4.5MB 벽 ❌ (Vercel Blob 없으면 막힘)
[처리 시간 10~30초] → 10초 제한 ❌ (Pro 플랜 없으면 타임아웃)
```

**결론**: 80페이지 이상은 Pro 플랜 ($20/월) + Vercel Blob 설정 필수

## 🔧 그래서 어떻게 만들면 좋나요?

### 추천 방식: Node.js + Vercel Blob

**🟢 초딩용 설명**:  
파이썬 도구(pdfplumber) 대신 자바스크립트 도구(unpdf)를 써요.  
자바스크립트 도구는 Vercel에서 잘 작동하도록 만들어졌거든요.  
PDF 뽑아내는 정확도는 조금 떨어지지만, 안정적으로 작동해요.

**🔵 개발자용 설명**:  
`unpdf` 라이브러리는 순수 JS, 네이티브 의존성 없음.  
Vercel 번들러와 100% 호환. 서버리스 환경 공식 지원.  
테이블 추출 정확도는 pdfplumber 대비 70~80% 수준이나, 건강보험공단 정형화된 PDF에서는 충분할 가능성 높음.

---

### 단계별 시작 전략

```
📌 Phase 1: 무료로 시작 (Hobby 플랜)
   → PDF 크기 4.5 MB 이하 파일만 처리
   → 소수 페이지 PDF 위주
   → 비용: $0
   → 목적: 기능 검증

📌 Phase 2: Pro 플랜 전환 ($20/월)
   → 80페이지 이상 PDF 처리
   → Vercel Blob으로 대용량 업로드
   → 실무 보험설계사 환경
   → 비용: 월 $20~25

📌 Phase 3 (선택): 정확도 높이기
   → Python pdfplumber를 별도 서버(Railway 등)에 올림
   → 추가 비용 월 $0~7 (소규모)
   → 정확도 최고 수준
```

## ✅ 오늘 기억할 것 3가지

1. **Vercel 무료 플랜** = 10초 제한 + 4.5MB 제한 → 작은 PDF만 가능

2. **80페이지 이상 PDF** = 반드시 Pro 플랜 ($20/월) 필요

3. **Python pdfplumber** = 정확하지만 Vercel에서 불안정 → 초기에는 Node.js 라이브러리로 시작

## ❓ 자주 물어볼 것 같은 질문

**Q. 무료로는 아무것도 못 하나요?**  
A. 아니요. 작은 PDF(10페이지 이하, 4.5MB 미만)는 무료로 처리 가능해요. 처음 개발하고 테스트하는 동안은 무료로 충분해요.

**Q. Pro 플랜 $20이면 너무 비싼 거 아닌가요?**  
A. 보험설계사 1명이 하루 30분씩 수작업 절감되면 월 수십만 원 가치예요. $20(약 2.8만 원)이면 충분히 합리적이에요.

**Q. 고객 개인정보가 Vercel 서버를 거쳐도 괜찮나요?**  
A. Vercel은 미국 회사입니다. 의료 정보를 해외 서버에 올리는 것이 법적으로 문제없는지 확인이 필요해요. (HIPAA는 미국 기준, 한국은 개인정보보호법 적용)  
대안: 브라우저에서만 처리하는 방식 — 외부 서버 없음, 개인정보 유출 위험 없음 (하지만 개발이 복잡해짐)

**Q. 건강보험공단 PDF가 언제 양식을 바꾸면?**  
A. 자동 파서가 깨집니다. 양식 버전 감지 로직을 나중에 추가해야 해요.

# 초등학생도 이해하는 로컬 배포 설명서

> 보험설계사 PC에 직접 설치해서 쓰는 버전, 왜 필요한지 + 어떻게 만드는지

---

## 🌐 클라우드 vs 로컬, 뭐가 다른가요?

### 클라우드 방식 (Vercel)

```
설계사 PC                      인터넷                    Vercel 서버 (미국)
   │                              │                           │
[PDF 선택] ──── 파일 전송 ───────────────────────────→ [처리]
                                                             │
[결과 확인] ←───────────────────────────── 결과 전송 ────────┘
```

- 인터넷만 되면 어디서든 접속 가능
- 설치 필요 없음 (브라우저에서 바로)
- **단점**: 파일이 미국 서버를 거쳐감, 크기 제한 있음, 돈 들 수 있음

---

### 로컬 방식

```
설계사 PC
   │
[PDF 선택]
   │
[내 PC에서 처리] ← 인터넷 전혀 필요 없음
   │
[결과 확인]
```

- 모든 처리가 내 PC 안에서만 일어남
- 파일이 외부로 나가지 않음 → 개인정보 완전 보호
- 파일 크기 제한 없음, 시간 제한 없음
- **단점**: 설치가 필요함

---

## 🏠 로컬 버전 3가지 방법

### 방법 1: 파이썬 서버 방식 (개발자가 쓰기 좋음)

**쉬운 설명**:  
컴퓨터 안에 작은 서버를 직접 켜는 방식이에요.  
마치 집에 있는 공유기처럼 — 내 컴퓨터가 서버 역할을 해요.

```
설계사가 할 일:
1. 프로그램 설치 (한 번만)
2. 아이콘 더블클릭 (매번)
3. 브라우저에서 localhost:8000 입력 (매번)
4. PDF 업로드하고 결과 받기
```

**장점**
- ✅ PDF 크기 제한 없음 (80페이지도 OK)
- ✅ 처리 시간 제한 없음
- ✅ 인터넷 없어도 됨
- ✅ 개인정보 외부 유출 없음
- ✅ 비용 $0

**단점**
- ❌ "localhost:8000 입력" 이 부분이 일반인에겐 낯섦
- ❌ 파이썬 설치 필요 (처음 한 번)
- ❌ PC마다 환경이 달라서 "내 컴퓨터에선 안 돼요" 문제 가능

**비유**: 집에서 직접 밥 해먹기 → 재료 사오는 게 귀찮지만, 먹을 때는 식당보다 훨씬 자유로움

---

### 방법 2: 도커 방식 (안정적이지만 설치 어려움)

**쉬운 설명**:  
도커(Docker)는 "프로그램이 항상 같은 환경에서 실행되도록 포장하는 상자"예요.  
택배 상자처럼 — 어떤 컴퓨터에서 열어도 안에 있는 내용물이 똑같아요.

```
설계사가 할 일:
1. Docker Desktop 설치 (좀 복잡, 처음 한 번)
2. 명령어 한 줄 입력: docker run medi-data-nexus
3. 브라우저에서 localhost:8000 접속
4. PDF 업로드하고 결과 받기
```

**장점**
- ✅ "내 컴퓨터에선 안 돼요" 문제 없음 (환경 완전 통일)
- ✅ PDF 크기·시간 제한 없음
- ✅ 개인정보 외부 유출 없음

**단점**
- ❌ Docker Desktop 자체가 설치 어려움 (설정 필요, 500MB)
- ❌ 일반 설계사에겐 너무 복잡
- ❌ PC 메모리를 많이 먹음

**비유**: 에어프라이어 구매 → 요리 결과는 항상 일정하지만, 처음 설치하고 사용법 익히는 게 좀 번거로움

---

### 방법 3: 설치 파일(.exe) 방식 ← 최종 목표

**쉬운 설명**:  
카카오톡처럼 설치 파일을 받아서 더블클릭하면 바로 쓸 수 있는 방식이에요.

```
설계사가 할 일:
1. setup.exe 받아서 더블클릭 (처음 한 번, 끝)
2. 바탕화면 아이콘 더블클릭
3. 앱 창 열리면 PDF 드래그&드롭
4. 결과 확인, 엑셀 다운로드
```

**장점**
- ✅ 카카오톡 설치하는 것과 완전히 동일한 경험
- ✅ 파이썬, 도커 전혀 몰라도 됨
- ✅ PDF 크기·시간 제한 없음
- ✅ 완전 오프라인 실행
- ✅ 개인정보 완전 보호

**단점**
- ❌ 만들기가 제일 복잡함 (개발 공수 많음)
- ❌ 파일 크기 150~300MB (설치 파일이 큼)
- ❌ 업데이트 배포할 때마다 새 파일 만들어야 함

**비유**: 즉석밥 → 사용자 입장에서 가장 편하지만, 만드는 공장 입장에선 포장·유통이 복잡함

## 🤔 "클라우드 vs 로컬" 어떤 걸 써야 하나요?

### 클라우드(Vercel)가 맞는 경우
- 외근 중 스마트폰이나 다른 PC에서 써야 할 때
- 설치 없이 바로 쓰고 싶을 때
- PDF가 4.5MB 이하이고 간단한 문서일 때

### 로컬이 맞는 경우
- **80페이지 이상 두꺼운 PDF** 처리할 때 ← 이게 주요 이유
- 고객 의료 정보를 외부 서버에 보내기 껄끄러울 때
- 인터넷이 없거나 느린 환경일 때
- 비용을 $0으로 유지하고 싶을 때

## ❓ 자주 물어볼 것 같은 질문

**Q. 설계사 PC에 파이썬을 설치하게 하면 너무 어렵지 않나요?**  
A. 맞아요. 그래서 최종 버전은 설치파일(.exe)로 만드는 게 목표예요.  
개발 초기에는 파이썬 서버 방식으로 내부 테스트하고, 안정화되면 .exe로 포장해요.

**Q. 클라우드와 로컬을 동시에 제공할 수 있나요?**  
A. 네! 같은 UI에서 "클라우드 모드 / 로컬 모드" 토글 버튼을 만들면 돼요.  
인터넷 될 때는 클라우드, 오프라인이거나 큰 파일은 로컬로 자동 전환도 가능해요.

**Q. 로컬 버전 업데이트는 어떻게 하나요?**  
A. .exe 방식이면 "자동 업데이트" 기능을 넣어야 해요.  
앱 실행 시 새 버전 있으면 "업데이트하시겠어요?" 팝업 → 클릭 → 자동 설치.  
이 기능은 Phase 3에서 구현해요.

**Q. Mac 쓰는 설계사는요?**  
A. Windows .exe는 Mac에서 안 열려요.  
Mac용은 .dmg 파일을 따로 빌드해야 해요.  
Phase 3에서 Windows 먼저 만들고 Mac은 수요 확인 후 결정해요.
